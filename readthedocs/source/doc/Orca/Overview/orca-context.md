    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-orca-context" class="anchor" aria-hidden="true" href="#orca-context"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Orca Context</h1>
<hr>
<p><code>OrcaContext</code> is the main entry for provisioning the Orca program on the underlying cluster (such as K8s or Hadoop cluster), or just on a single laptop.</p>
<hr>
<h3><a id="user-content-1-initialization" class="anchor" aria-hidden="true" href="#1-initialization"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><strong>1. Initialization</strong></h3>
<p>An Orca program usually starts with the initialization of <code>OrcaContext</code> as follows:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">from</span> <span class="pl-s1">zoo</span>.<span class="pl-s1">orca</span> <span class="pl-k">import</span> <span class="pl-s1">init_orca_context</span>

<span class="pl-en">init_orca_context</span>(...)</pre></div>
<p>In <code>init_orca_context</code>, the user may specify necessary runtime configurations for the program, including:</p>
<ul>
<li><em>Cluster mode</em>: Users can specify the computing environment for the program (a local machine, K8s cluster, Hadoop/YARN cluster, etc.).</li>
<li><em>Physical resources</em>: Users can specify the amount of physical resources to be allocated for the program on the underlying cluster, including the number of nodes in the cluster, the cores and memory allocated for each node, etc.</li>
</ul>
<p>The Orca program simply runs <code>init_orca_context</code> on the local machine, which will automatically provision the runtime Python environment and distributed execution engine on the underlying computing environment (such as a single laptop, a large K8s or Hadoop cluster, etc.). &lt;TODO: Add a architecture chart?&gt;</p>
<p>View the related <a href="/intel-analytics/analytics-zoo/blob/master/docs/readthedocs/Orca/Overview">Python API doc</a> for more details.</p>
<hr>
<h3><a id="user-content-2-python-dependencies" class="anchor" aria-hidden="true" href="#2-python-dependencies"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><strong>2. Python Dependencies</strong></h3>
<p>A key challenge for scaling out Python program across a distributed cluster is how to properly install the required Python environment (libraries and dependencies) on each node in the cluster (preferably in an automatic and dynamic fashion).</p>
<p>For K8s cluster, the user may install required Python packages in the container and specify the <code>container_image</code> argument when <code>init_orca_context</code>. For Hadoop/YARN cluster, the user may use <code>conda</code> to create the Python virtual environment with required dependencies on the local machine, and <code>init_orca_context</code> will automatically detect the active <code>conda</code> environment and provision it on each node in the cluster.</p>
<p>View the user guide for <a href="/intel-analytics/analytics-zoo/blob/master/docs/readthedocs/Orca/Overview">K8s</a> and <a href="/intel-analytics/analytics-zoo/blob/master/docs/readthedocs/Orca/Overview">Hadoop/YARN</a> for more details.</p>

<h3><a id="user-content-4-extra-configurations" class="anchor" aria-hidden="true" href="#4-extra-configurations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a><strong>4. Extra Configurations</strong></h3>
<p>Users can make extra configurations when using the functionalities of Project Orca via <code>OrcaContext</code>.</p>
<ul>
<li><code>OrcaContext.log_output</code>: Default to be False. Setting it to True is recommended when running Jupyter notebook (this will display all the program output in the notebook). Make sure you set it before <code>init_orca_context</code>.</li>
<li><code>OrcaContext.serialize_data_creator</code>: Default to be False. Setting it to True would add a file lock when initializing data for distributed training (this may be useful if you run multiple workers on a single node and they download data to the same destination).</li>
<li><code>OrcaContext.pandas_read_backend</code>: Setting it to the backend to be used for reading data as Panda DataFrame. See <a href="https://github.com/intel-analytics/analytics-zoo/blob/master/docs/readthedocs/Orca/Overview/data-parallel-processing.md#31-data-parallel-pandas">here</a> for more details.</li>
<li><code>OrcaContext.train_data_store</code>: Setting memory type for train data storage. Either <code>DRAM</code>, <code>PMEM</code>, or <code>DISK_n</code>. Default to <code>DRAM</code>; setting it to <code>DISK_n</code> (e.g., <code>DISK_2</code>) if the training data cannot fit in memory (this will store the data on disk, and cache only <code>1/n</code> of the data in memory; after going through the <code>1/n</code>, we will release the current cache, and load another <code>1/n</code> into memory). You can set it to <code>PMEM</code> if have AEP hardware.</li>
</ul>
